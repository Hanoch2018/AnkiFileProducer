# ==========================================
# 802.11 MAC 体系结构
# ==========================================
Basic ||| 1数据字节由8bit组成，数据字节又称为什么？ ||| 八比特组（octet）

802.11数据链路分为两个子层。上层是802.2逻辑链路控制(LLC)子层，所有802网络具有相同的LLC子层结构，但并非所有的802网络都使用LLC子层。
下层是介质访问控制(MAC)子层，802.11标准定义的无线局域网通信位于MAC子层。





# ==========================================
# 802.11 基础概念与帧结构
# ==========================================
Basic ||| 802.11 MAC 帧主要分为哪三大类？ ||| 管理帧 (Management)、控制帧 (Control)、数据帧 (Data)。 ||| 备注：例如 Beacon 属于管理帧，ACK 和 RTS/CTS 属于控制帧。
Cloze ||| 802.11 设备的连接过程通常包含三个状态。状态 1：未认证且未关联；状态 2：{{c1::已认证 (Authenticated)}} 但未关联；状态 3：已认证且 {{c2::已关联 (Associated)}}。 ||| 只有达到状态 3，STA 才能与 AP 进行数据帧的传输。
Basic ||| 在 802.11 抓包中，BSSID 通常代表什么？ ||| 通常代表 AP (Access Point) 的 MAC 地址，用于唯一标识一个基本服务集 (BSS)。 ||| 

# ==========================================
# 功耗管理与 802.11ax 特性
# ==========================================
Cloze ||| 802.11ax 引入了核心机制 {{c1::TWT (Target Wake Time)}}，允许 AP 和 STA 协商具体的唤醒时间，从而极大地降低终端设备的功耗。 ||| 备注：在进行功耗抓取和分析（如使用高精度功耗仪）时，这是非常关键的测试点。
Basic ||| Beacon 帧中的 TIM (Traffic Indication Map) 字段有什么作用？ ||| AP 通过 TIM 通知处于休眠 (Sleep) 状态的 STA，AP 的缓存中是否有发往该 STA 的单播数据。 ||| STA 醒来收完 Beacon 发现 TIM 指示有自己的数据后，会发送 PS-Poll 帧去取数据。
Basic ||| 什么是 DTIM (Delivery Traffic Indication Map)？ ||| 它是 TIM 的一种特殊类型，用于通知 STA 组播或广播数据即将发送。 |||

Basic ||| 如何在不丢失代码修改的前提下，让 Gerrit 生成一个新的 Change-Id？ ||| 执行 git commit --amend，在编辑器中手动删除现有的 Change-Id 行并保存。 ||| 备注：前提是本地已安装 Gerrit 的 commit-msg hook。
Basic ||| git reset --soft HEAD~1 的作用是什么？ ||| 撤销最近的一次提交（Commit），但保留该提交所做的代码改动在暂存区（Staged status）中。 ||| 备注：常用于合并提交或重新生成 Change-Id。

# Linux-Unix系统编程手册
Basic ||| 在狭义的操作系统定义中，“内核 (Kernel)”的主要职责是什么？ ||| 管理和分配计算机资源，包括 CPU、RAM 和外部设备。 |||
Basic ||| Linux 内核文件通常命名为 /boot/vmlinuz，其中末尾的 "z" 代表什么？ ||| 代表该内核是一个经过压缩 (Compressed) 的可执行文件。 ||| 备注：名称演变路径为 unix -> vmunix -> vmlinuz。
Basic ||| 什么是“抢占式多任务 (Preemptive Multitasking)”？ ||| 由内核的进程调度程序（而非进程本身）决定哪些进程获得 CPU 使用权以及使用时长。 |||
Basic ||| 虚拟内存 (Virtual Memory) 管理机制的两个主要优势是什么？ ||| 1. 进程间及进程与内核间的内存隔离；2. 只需将进程的一部分保持在内存中，提高内存利用率。 |||
Basic ||| 处理器在“用户态 (User Mode)”和“核心态 (Kernel Mode)”下的权限有什么区别？ ||| 在用户态下，CPU 只能访问用户空间内存且不能执行特权指令；在核心态下，CPU 可以访问所有内存并执行所有硬件指令。 ||| 备注：这种机制确保了系统的安全性和稳定性。
Basic ||| 系统调用 (System Call) 的作用是什么？ ||| 它是受控的内核入口点，允许进程请求内核执行特定的任务（如 I/O 操作）。 |||
Basic ||| 在 Linux/UNIX 系统中，识别用户身份的两个唯一标识是什么？ ||| 登录名 (Login Name) 和 用户 ID (UID)。 ||| 备注：UID 是一个整数。
Basic ||| 系统密码文件 /etc/passwd 中，每个用户记录包含哪些核心信息？ ||| 1. 登录名；2. 用户 ID (UID)；3. 组 ID (GID)；4. 主目录 (Home directory)；5. 登录 shell。 ||| 备注：该文件过去也存储加密密码。
Basic ||| 出于安全考虑，现代系统通常将用户密码存储在哪个文件中？ ||| /etc/shadow。 ||| 备注：该文件仅供特权用户（如 root）阅读。
Basic ||| 在 Linux 系统中，用户和“组”的关系是怎样的？ ||| 一个用户可以同时隶属于多个组。 ||| 备注：目的是为了控制对文件和其他资源的访问（权限管理）。
Basic ||| 超级用户 (Superuser) 的登录名和 UID 分别是什么？ ||| 登录名为 root，用户 ID (UID) 为 0。
Basic ||| 系统组文件 /etc/group 的每行记录包含哪些信息？ ||| 1. 组名 (Group Name)；2. 组 ID (GID)；3. 用户列表 (User List)。 ||| 备注：用户列表包含该组中除初始组成员外的其他成员登录名。
Basic ||| 超级用户 (root) 在系统权限方面有哪些特殊性？ ||| 1. 凌驾于系统权限检查之上；2. 可以访问系统中的任何文件；3. 可以发送信号干预任何用户进程。 ||| 备注：系统管理员使用该账号执行维护任务。

Basic ||| Linux/UNIX 的目录层级结构与 Windows 有何本质区别？ ||| Linux 采用 单根目录层级（以 / 为起点），所有文件都在此层级下；Windows 的每个磁盘设备有各自的目录层级。 ||| 备注：Windows 使用驱动器盘符（如 C:、D:）。
Basic ||| 在 Linux 系统编程中，术语“文件” (File) 包含哪些类型？ ||| 1. 普通文件 (Regular file)；2. 目录 (Directory)；3. 设备 (Device)；4. 管道 (Pipe)；5. 套接字 (Socket)；6. 符号链接 (Symbolic link)。 ||| 备注：即“一切皆文件”的概念。
Basic ||| 什么是“链接” (Link)？ ||| 目录中的一条记录，由 文件名 + 对相应文件的引用 组成。 ||| 备注：每个文件可以有多个硬链接，从而在不同目录中有不同名称。
Basic ||| 根目录的父目录条目 (/..) 指向哪里？ ||| 指向 根目录自身（即 /.. 等于 /）。
Basic ||| 符号链接 (Symbolic Link) 的内容包含什么？ ||| 经过特殊标记的文件，其内容是 另一个文件的名称（路径字符串）。 ||| 备注：又称软链接 (Soft Link)。
Basic ||| 什么是“空链接” (Dangling Link)？ ||| 指符号链接所指向的目标文件 并不存在。 ||| 备注：也常被称为死链接。
Basic ||| Linux 文件名命名时，哪两个字符是绝对禁用的？ ||| 正斜杠 (/) 和 空字符 (\0)。 ||| 备注：/ 用作路径分隔符，\0 用于标识字符串结束。
Basic ||| 为什么不建议以连字符 (-) 作为文件名的起始字符？ ||| 因为在 shell 命令中使用该文件名时，会被误认为是 命令行选项开关 (Options/Flags)。
Baisc ||| 每个文件都有什么属性分别定义其的属主和属组？ ||| 用户ID和组ID
Basic ||| 为了访问文件，系统把用户分成了三类，分别是？ ||| 文件的属主（有时也称为文件的用户），与文件组ID相匹配的属组成员用户以及其他用户

Reversed ||| Hard link ||| 硬链接 ||| 备注：目录中内容为“文件名 + 指针”的正常记录。
Reversed ||| Portable filename character set ||| 可移植文件名字符集 ||| 备注：由 SUSv3 定义，包含 [-._a-zA-Z0-9] 共 65 个字符。

Basic ||| Linux 内核如何理解文件的本质？ ||| 内核将文件视为 字节流序列 (Sequence of bytes)，不关心其内部结构。 ||| 备注：结构化解析由应用层负责。
Basic ||| 在文件 I/O 中，lseek() 系统调用的核心作用是什么？ ||| 显式地修改 当前文件偏移量 (Current file offset)，从而实现 随机访问。 ||| 备注：可以跳转到文件的任意位置读写。
Basic ||| 什么是文件描述符 (File Descriptor)？ ||| 一个（通常较小的）非负整数，用于在 I/O 系统调用中指代打开的文件。 ||| 备注：它是进程访问文件的“句柄”。
Basic ||| 在系统编程中，获取文件描述符最常用的手法是什么？ ||| 调用 open() 系统调用，并指定目标文件的路径名。 ||| 备注：其他如 pipe() 或 socket() 也能获取。
Basic ||| 请列出进程默认继承的三个文件描述符及其含义。 ||| 1. 0：标准输入 (Standard Input)；2. 1：标准输出 (Standard Output)；3. 2：标准错误 (Standard Error)。 ||| 备注：这些描述符通常在交互式 shell 中指向终端。
Basic ||| 文件描述符 0, 1, 2 分别对应 stdio 函数库中的哪些文件流？ ||| 0 对应 stdin；1 对应 stdout；2 对应 stderr。 ||| 备注：stdio 是在系统调用层之上的封装。
Basic ||| 在交互式程序中，标准输入、输出和错误默认指向哪个设备？ ||| 终端 (Terminal)。 ||| 备注：可以通过重定向更改指向。

Basic ||| Python subprocess 模块捕获输出的底层机制，主要依赖于哪种进程间通信 (IPC) 方式？ ||| 依赖于 管道 (Pipe)。 ||| 备注：通常配合 fork() 和 exec() 使用。
Basic ||| 在 Linux 系统编程中，常用于实现“文件描述符重定向”（如重定向 stdout）的核心系统调用是什么？ ||| dup2() (或 dup())。 ||| 备注：它可以将一个旧的文件描述符（如 1）指向一个新的已打开文件或管道。
Basic ||| 当我们在 shell 或 Python 脚本中将标准输出重定向到文件时，底层程序的代码需要修改吗？为什么？ ||| 不需要修改。因为程序只负责向固定的文件描述符 (FD 1) 写入字节流序列，具体 FD 1 指向屏幕、文件还是管道，由操作系统内核负责调度，对应用层是透明的。

Basic ||| UNIX 系统 I/O 模型中最显著的特性“I/O 通用性”是指什么？ ||| 指的是使用同一套系统调用（如 open()、read()、write()、close()）可以对所有文件类型（包括设备文件）执行 I/O 操作。 ||| 备注：内核将应用程序的请求转化为相应的文件系统或设备驱动程序操作。
Basic ||| 在 UNIX 系统中，如何判断已经到达了文件末尾 (EOF)？ ||| UNIX 系统没有专门的文件结束符。读取文件时，如果没有数据返回（返回值为 0），则认定抵达文件末尾。 ||| 备注：这与某些系统使用特定字符作为 EOF 的做法不同。

Basic ||| 什么是“过滤器” (Filter) 程序？请举例。 ||| 指从标准输入 (stdin) 读取数据，经过转换后输出到标准输出 (stdout) 的程序。如：cat、grep、tr、sort、wc、sed、awk。 ||| 备注：这是 UNIX 管道哲学的重要组成部分。
Basic ||| C 语言 main 函数声明中，argc 和 argv 分别代表什么？ ||| argc：命令行参数的总个数；argv：指向参数字符串的指针数组，其中 argv[0] 是程序名本身。 ||| 备注：这是程序获取外部输入的基本方式。

Basic ||| 简述进程 (Process) 的四个主要内存段 (Segments)。 ||| 1. 文本段 (Text)：程序的机器指令。 2. 数据段 (Data)：静态变量。 3. 堆 (Heap)：动态分配的额外内存。 4. 栈 (Stack)：局部变量和函数调用链接信息。 ||| 备注：理解这些段对调试 C 语言内存泄露或溢出非常重要。
Basic ||| 系统调用 fork() 和 execve() 在创建进程/执行程序时有什么区别？ ||| fork()：创建一个子进程，它是父进程的副本（复制数据、栈、堆段）；execve()：销毁当前进程的内存段，并加载并执行一个全新的程序。 ||| 备注：通常先用 fork() 创建进程，再在子进程中调用 exec()。
Basic ||| 进程标识符中，PID 和 PPID 分别指什么？ ||| PID (Process ID)：每个进程唯一的整数标识符。PPID (Parent Process ID)：标识创建该进程的父进程。 ||| 备注：可以使用 getpid() 和 getppid() 系统调用获取。
Reversed ||| Bookkeeping ||| （内核）记账 ||| 备注：内核建立 bookkeeping 数据结构来记录 PID、UID、终止状态等进程信息。
Basic ||| 为什么说 UNIX 系统中“一切皆文件”对测试脚本编写很有利？ ||| 因为程序员可以使用统一的 read() 和 write() 处理磁盘文件、终端、甚至 Wi-Fi 设备驱动。 ||| 备注：这实现了 I/O 通用性 (I/O Generality)。
Basic ||| C 语言 stdio 函数库（如 printf, fopen）与系统调用（如 write, open）是什么关系？ ||| stdio 函数库位于系统调用层之上。 ||| 备注：stdio 封装了系统调用以提供更好的可移植性和带缓冲的 I/O 效率。
Basic ||| 为什么 init 进程被称为“所有进程之父”？ ||| 它是系统启动时内核创建的第一个进程（PID 为 1），系统中所有其他进程要么由它创建，要么是其后代。 ||| 备注：/sbin/init 以超级用户权限运行，且不能被手动杀掉。

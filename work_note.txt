
Basic ||| iperf3 进行反向测试（测试下行带宽）的参数是什么？||| -R ||| 备注：默认是客户端发送、服务端接收，加上 -R 后改为服务端发送、客户端接收。
Basic ||| iperf3 测试 UDP 时，除了带宽（Bitrate）外，还可以观察哪两个关键网络指标？ ||| 抖动 (Jitter) 和 丢包率 (Lost/Total Datagrams)。 ||| 备注：UDP 测试通常需要用 -b 指定目标带宽。
Basic ||| iperf3 测试时想要一直打流，可以用什么参数？ ||| -t 0 ||| 
Basic|||iperf3 中 -P 参数的作用是什么？|||设置并行（Parallel）客户端流的数量。|||备注：用于验证多流并发下的系统总带宽。
Basic ||| iperf3 测试时想要输出JSON格式的结果，可以用什么参数？ ||| -J ||| 方便自动化测试解析结果
Basic||| 在 iperf3 测试中，参数 -O 的作用是什么？||| 忽略（Omit）测试开始前几秒的数据。||| 备注：用于排除 TCP 慢启动或系统初始波动对平均速率的影响。
Basic|||iperf3 如何进行下行带宽测试（由服务端发送，客户端接收）？|||使用 -R 参数。|||备注：即 Reverse 反向模式，默认测试方向是客户端发、服务端收。
Basic|||iperf3 进行 UDP 测试时，默认带宽是多少？|||1 Mbps。|||备注：测试 UDP 时必须用 -b 参数手动指定目标带宽。
Basic|||进行高带宽（如 10Gbps）iperf 测试时，为什么必须监测 CPU 占用率？|||防止 CPU 满载成为瓶颈。|||备注：如果 CPU 占用接近 100%，测得的是处理器的极限而非真实网络带宽。

Basic ||| 如何在不丢失代码修改的前提下，让 Gerrit 生成一个新的 Change-Id？ ||| 执行 git commit --amend，在编辑器中手动删除现有的 Change-Id 行并保存。 ||| 备注：前提是本地已安装 Gerrit 的 commit-msg hook。
Basic ||| git reset --soft HEAD~1 的作用是什么？ ||| 撤销最近的一次提交（Commit），但保留该提交所做的代码改动在暂存区（Staged status）中。 ||| 备注：常用于合并提交或重新生成 Change-Id。
Basic ||| GitHub 允许上传的单个文件最大是多少？ ||| 100 MB。超过此限制会导致 pre-receive hook declined 错误。 ||| 备注：超过 50MB 时 GitHub 就会发出警告。
Basic ||| Git LFS 的全称和主要作用是什么？ ||| 全称：Git Large File Storage。作用：通过将大文件替换为文本指针，将文件内容存储在远程服务器上，解决 Git 仓库处理大文件效率低和托管平台大小限制的问题。 ||| 备注：常用命令包括 git lfs track。
Basic ||| 为什么先 commit 了大文件，再执行 git lfs track 依然无法 push？ ||| 因为大文件已经以普通 Git 对象的形式存在于历史 Commit 中。LFS 规则仅在执行 git add 时触发。 ||| 备注：需要 reset 撤销提交后重新 add 才能生效。
Basic ||| 如何查看当前仓库中哪些文件正由 Git LFS 管理？ ||| 使用命令 git lfs ls-files。 ||| 备注：这可以确认大文件是否已成功转换为 LFS 指针。
